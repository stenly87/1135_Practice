Задание на практику:
Разбиваетесь на группы 2-3 человека
1 группа делает сервер приложения
Остальные - клиент приложение

Приложение:
asp net + signalr
каждый клиент - веб-приложение, которое получает по signalr уведомления
от сервера и может слать на сервер такие же уведомления (тоже signalr)

На сервере хранится чат, каждый клиент - участник чата, который 
может вывести чат на экран

При этом клиенты чатятся самостоятельно, т.е. на каждом компьютере
настроена локальная нейронка (очень желательно, разная)

Плюс должна быть возможность закинуть в чат анонимное сообщение
 с любого клиента (т.е. не от лица нейронки)
 
Сервер - после после получения сообщения, он добавляет его в конец чата,
берет последних 20-30 сообщений и отправляет одному из клиентов,
 остальным просто шлет уведомления о новом сообщении
 
Клиент, который получил 20-30 сообщений генерирует ответ с помощью
локальной нейронки

Каждая нейронка может быть настроена по вашему вкусу (т.е. системный
промт у всех должен быть разный)

Можно использовать LMStudio или аналог. Это приложение позволяет 
скачать модель с https://huggingface.co/ (берите модели,
 генерирующие текст, на 4B или 8B аргументов)
 
LMStudio (при настройке) поднимает локальный REST-api, куда можно
 отправлять запросы загруженной модели
 
Принимаем ответ от нейронки, форматирует для отправки на сервер,
отправляем
